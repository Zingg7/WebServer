WebServer_v8

本版本中完成响应客户端404页面的工作

当客户端请求的资源没有找到时,我们应当响应客户端404页面
404页面是一个公共页面,不属于某个具体的webapp,无论用户请求那个网络应用(webapp)的资源时
只要不存在都应当响应这个页面(后期可以继续完善,使得每个应用都有自己的404)

实现:
1) 在webapp目录中新建一个子目录:root
2) 在root目录中新建页面:404.html
3) 由于响应404页面的同时还要将当前响应中状态行中状态代码 与描述改为404,NOT FOUND
  因此我们要将HttpResponse中发送状态行的操作进行修改,不能发送固定的200 OK,应当改为可以进行设置的
	3.1) 在HttpResponse中添加两个新的属性
   	 int statusCode = 200;
   	 String statusReason = "OK"
    	并且赋予它们默认值200,OK.因为大部分的请求我们都应当正确处理,所以正确的就无须再设置这两项
	3.2) 提供者两个属性的get,set方法,工外界可以进行设置
	3.3) 修改sendStatusLine方法,将状态行内容中状态代码与描述部分改为使用者两个属性进行发送
4) 在ClientHandler的处理请求分支中,当资源没有找到时,设置response响应404页面,同时设置好对应的状态代码与状态描述:404 NOT FOUND
